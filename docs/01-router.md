# doido-router — Spec

Rails analogue: **Action Dispatch / Router**

## Decisions (resolved in interview)

- **Route definition style:** Macro DSL — primary and only API
- **`resources!` generates all 7 REST routes by default**, with `only:` / `except:` options to restrict
- Named route URL helpers generated by the macro (compile-time)

## Macro DSL Design

```rust
routes! {
    resources!(posts, PostsController);
    resources!(comments, CommentsController, only: [index, show]);
    resources!(admin, AdminController, except: [destroy]);

    get!("/about", PagesController::about);
    post!("/login", SessionsController::create);

    namespace!(api, {
        resources!(users, Api::UsersController);
    });

    scope!("/v2", {
        resources!(articles, V2::ArticlesController);
    });
}
```

## 7 REST Routes generated by `resources!(name, Controller)`

| Helper          | Method | Path                  | Action    |
|-----------------|--------|-----------------------|-----------|
| `posts_path()`  | GET    | /posts                | index     |
| `new_post_path()` | GET  | /posts/new            | new       |
| `post_path(id)` | GET    | /posts/:id            | show      |
| `post_path(id)` | POST   | /posts                | create    |
| `edit_post_path(id)` | GET | /posts/:id/edit    | edit      |
| `post_path(id)` | PATCH  | /posts/:id            | update    |
| `post_path(id)` | DELETE | /posts/:id            | destroy   |

## Known Requirements

- Built on top of `axum::Router` internally
- Routes map to controller actions (not raw closures)
- Must support: GET, POST, PUT, PATCH, DELETE, HEAD, OPTIONS
- Route parameters extracted and typed via serde
- URL helpers generated by the `routes!` macro at compile time
- `namespace!` prefixes both path and module path
- `scope!` prefixes only the path
